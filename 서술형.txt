1.	동기와 비동기 처리에 대하여 설명하라.
동기 처리 : 서버에 요청을 보냈을 때 응답이 와야 처리가 가능한 방식
비동기 처리 : 서버에 요청을 보냈을 때 응답이 없어도 처리를 할수 있는 방식
2.	싱글 스레드와 멀티 스레드에 대하여 설명하라.
싱글 스레드 : 한 개의 노드를 처리 할수 있는 쓰레드
멀티 스레드 : 여러 개의 노드를 동시에 처리 할수 있는 쓰레드
3.	쿠키와 세션, 로컬저장소의 차이점에 대하여 설명하라.
쿠키: 클라이언트에 키와 값이 저장되고 브라우저를 종료해도 인증이 유지됨 헤더에 쿠키를 포함시킴
세션 : 쿠키와 비슷하지만 세션은 서버에서 관리
로컬 저장소 : 자신이 코딩하고 있는 컴퓨터 
4.	콜백지옥이 무엇이고 이것을 해결하기 위한 방안은 무엇인지 설명하라.
콜백지옥: 콜백 함수를 쓸 때 안에 있는 파라미터의 값을 이용해서 하게 되는데 그 과정에서 함수가 무한 반복되는 현상
5.	Node.js 이벤트 큐 동작원리에 대하여 설명하라.
Nodejs는 타이머 대기열에 있는 만기된 타이머가 있는지 먼저 확인하면서 이벤트 루프를 시작한다. 그리고 처리할 총 아이템의 카운터 참조를 유지하면서 각 단계에서 각 큐를 실행한다. 클로즈 핸들러 큐를 처리한 이후에, 더 이상 대기중인 큐가 없고 동작중인 작업이 없다면 루프를 빠져나가게 된다. 이벤트 루프에서 각 큐의 처리는 이벤트 루프의 한 단계로 볼수 있다.
6.	GET, POST 방식에 대하여 설명하라.
GET : 서버에서 데이터를 가져와서 보여줄 때 사용하는 방식
POST : 데이터를 가져온 것을 수정하고 삭제 할 때 사용하는 방식
7.	자바스크립트의 배열이 실제 자료구조 배열이 아닌데 그 이유는 무엇인가?
일반적인 배열에서는 데이터 공간이 일정하고 연속적으로 이어져있으나
자바스크립트에서의 배열은 데이터 공간이 일정하지 않아도 되고 연속적으로 이어져 있어야 된다는 조건이 없어서 실제 자료구조 배열이 아니다.
8.	http와 https의 차이점은 무엇인가?
http는 서버/클라이언트 모델을 따라 데이터를 주고 받기 위한 프로토콜이고 포트는 80번을 사용한다
반면에 https는 http에 데이터 암호화가 된 프로토콜이고 포트는 443번 포트를 사용한다
9.	브라우저 렌더링 과정에 대하여 설명하라.
브라우저에 대해 웹에서 페이지를 찾아서 보여주고 사용자가 하이퍼링크를 통해 다른 페이지로 이동할 수 있도록 하는 프로그램이다 브라우저는 유저가 선택한 자원을 서버에서 받아와서 유저에게 보여준다.
(
1.	파싱 html과 css파일을 파싱해서 각각 tree를 만든다.
2.	Style 두 tree를 결합해서 rendering tree를 만든다.
3.	Layout rendering tree에서 각 노드의 위치와 크기를 계산한다.
4.	Paint 계산된 값을 이용해 각 노드를 화면상의 픽셀로 전환하고, 레이어를 만든다.
5.	Composite 레이어를 합성하여 실제 화면에 나타낸다.
)
10.	호이스팅이 무엇인지 서술하라.
전역 변수를 모두 맨 위에서 선언하는것 함수 호이스팅(함수 선언문)도 가능하다.
